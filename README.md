# Парсер актуальных процедур ЭТП Газпромбанка

Цель проекта — автоматически собирать актуальные закупочные процедуры с площадки
[etpgaz.gazprombank.ru](https://etpgaz.gazprombank.ru/#com/procedure/index)
с авторизацией по ЭЦП (КриптоПро).

Сейчас реализован **Этап 1 — разведочный скрипт** `explore_etp.py`. Он
выполняет две задачи:

1. Проверяет, что на машине всё готово к работе с ЭЦП:
   - КриптоПро CSP
   - КриптоПро ЭЦП Browser plug-in (он же "Госплагин", CAdES)
   - Сертификаты в личном хранилище Windows (включая токены Рутокен/JaCarta)
   - Google Chrome и расширение *CryptoPro Extension for CAdES Browser*
   - Не запущен ли в момент запуска Chrome (он должен быть закрыт)
2. Запускает Chrome c Вашим основным профилем, открывает страницу
   актуальных процедур и записывает всё, что происходит на странице и в сети,
   в папку `exploration_results/`. Полученные файлы пересылаются разработчику,
   который на их основе собирает финальный парсер (Этап 2).

---

## Требования

- Windows 10/11
- Python 3.10+ (проверено на 3.13)
- Google Chrome (актуальная версия)
- Установленные КриптоПро CSP и CAdES Browser plug-in
- Расширение Chrome *CryptoPro Extension for CAdES Browser* в профиле **Default**

## Установка зависимостей

Откройте PowerShell в этой папке и выполните:

```powershell
pip install -r requirements.txt
```

## Запуск разведочного скрипта

1. **Закройте все окна Google Chrome** (скрипт использует Ваш основной
   профиль — он должен быть свободен). Проверить можно через Task Manager: не
   должно быть процессов `chrome.exe`.
2. Запустите скрипт:

   ```powershell
   python explore_etp.py
   ```

3. Скрипт сначала выведет результаты проверок окружения. Если критичные
   проверки провалились, он остановится и скажет, что нужно установить.
4. Если всё в порядке — откроется новое окно Chrome и загрузится страница
   `https://etpgaz.gazprombank.ru/#com/procedure/index`.
   В консоли появится подсказка: **"СЕЙЧАС ОКНО CHROME ОТКРЫТО..."**.
5. Пока окно открыто, сделайте в нём всё, что считаете нужным для разведки:
   - посмотрите страницу "Актуальные процедуры" (прокрутите вниз/вверх);
   - попробуйте нажать **"Вход по ЭЦП"** и пройти авторизацию (откроется
     окно выбора сертификата КриптоПро — выберите Ваш);
   - перейдите на вторую страницу реестра, смените фильтры;
   - откройте карточку любой процедуры.

   Все сетевые запросы, HTML и cookies будут автоматически записаны.
6. Когда закончите — вернитесь в консоль и нажмите **Enter**. Скрипт сохранит
   финальный слепок страницы и дампы сетевых логов, закроет браузер и
   сформирует отчёт.

## Где искать результаты

Все артефакты лежат в папке `exploration_results/`:

| Файл | Описание |
| --- | --- |
| `report.md` | Главный отчёт. Начинайте чтение с него. |
| `environment_checks.json` | Результаты всех проверок окружения. |
| `exploration_summary.json` | Сводка по разведке сайта. |
| `screenshot_initial.png` / `screenshot_final.png` | Скриншоты до/после Ваших действий. |
| `page_source_initial.html` / `page_source_final.html` | HTML страницы до/после. |
| `cookies_initial.json` / `cookies_final.json` | Cookies с площадки. |
| `buttons_and_links_*.json` | Все кнопки/ссылки/поля ввода на странице. |
| `dom_structure_*.json` | Верхнеуровневая структура DOM. |
| `storages_*.json` | localStorage и sessionStorage (полезно для API-токенов). |
| `network_log.json` | Все XHR/Fetch/Document-запросы с заголовками и телами. |
| `api_candidates.json` | Отфильтрованные XHR/Fetch-запросы только к etpgaz — это кандидаты на REST API. |
| `bodies/` | Полные тела ответов (если они были больше 3 КБ). |
| `browser_console_log.json` | Сообщения консоли браузера. |

### Что прислать разработчику

Содержимое всей папки `exploration_results/` можно упаковать в ZIP и прислать.
Критически важны:

- `report.md`
- `network_log.json`
- `api_candidates.json`
- `buttons_and_links_initial.json` и `buttons_and_links_final.json`

## Что будет дальше (Этап 2)

На основе собранных данных будет собран финальный парсер:

- если реестр процедур отдаётся открытым API, парсер будет использовать
  только `requests` без браузера (быстро и стабильно);
- если требуется вход по ЭЦП, парсер будет запускать Selenium, ждать пока Вы
  выберете сертификат в окне КриптоПро, а затем работать уже через API с
  сохранёнными cookies;
- результат (актуальные процедуры) будет выводиться в консоль и сохраняться в
  JSON.

## Частые ошибки

**"Chrome уже запущен"** — закройте все окна Chrome (включая процессы,
работающие в фоне). Скрипт специально предупреждает, если находит активные
`chrome.exe`.

**"user data directory is already in use"** — тоже значит, что Chrome открыт
или не успел корректно закрыться. Закройте/завершите процессы `chrome.exe`
через Task Manager и попробуйте ещё раз.

**"Не удалось открыть хранилище 'Мои' через CAdESCOM"** — не установлен
КриптоПро CAdES Browser plug-in. Скачайте
[отсюда](https://www.cryptopro.ru/products/cades/plugin).

**"Расширение CryptoPro Extension for CAdES Browser не найдено"** — установите
расширение в Chrome Web Store под аккаунтом, который соответствует профилю
`Default`.
